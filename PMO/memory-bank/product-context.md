# Product Context

## Why this Project Exists
This project is a high-fidelity frontend prototype of the **Corporate Public Mutual Online (PMO) Portal**. It is designed to demonstrate premium, cohesive, and modern user experiences (UX) and visual layouts for corporate investment management, account detail viewing, transaction approvals, and user onboarding.

Its primary purpose is to establish a state-of-the-art "Pro Max" design standard for Public Mutual's corporate systems, moving away from flat, outdated enterprise designs towards a premium, engaging, and highly trustworthy interface.

## User Experience Goals
- **Establish Trust:** A refined and clean aesthetic with bold, purposeful accents builds confidence in financial operations.
- **Visual Clutter Reduction:** Ample white space, generous container padding, and well-organized grids focus user attention on high-value tasks.
- **Fluid & Responsive Interaction:** Smooth state transitions, hover animations, and cohesive layouts make the portal feel tactile and professional.
- **Clear Information Hierarchy:** Important notifications, statuses, and call-to-actions are easily distinguishable using strategic semantic colors.

## Key Modules & Screens
1. **Login Portal (`index.html`)**: Features an elegant, offset right-aligned card with simulated loading states, password visibility toggles, and form clear capabilities.
2. **Dashboard (`unit-trust/dashboard.html`)**: The landing hub for corporate users (unit trust), displaying account summaries, quick links, pending tasks, and main navigation.
3. **Register (`register.html`)**: Multi-step corporate registration and onboarding workflow, highlighting user roles (e.g., Maker, Checker) during ID creation.
4. **Settings (`settings.html`)**: Configuration panel for security, profile management, and notification preferences.
5. **Transactions (`transactions.html`)**: Module listing pending and completed transaction requests with filtering and search.
6. **Authorise (`authorise.html`)**: Workspace for Checker roles to Authorise pending transactions.
7. **View Account Details (`view_account_detail.html` & `view_account.html`)**: Deep-dive screens for auditing specific corporate accounts, holdings, and histories.
8. **Top Up (`top_up.html`)**: Interface for making additional investment contributions.
9. **Analytics (`analytics.html`)**: Performance charts, asset allocation graphs, and visual indicators.
10. **Supporting Screens (`forgot.html`, `terms_conditions.html`, `internet_risk.html`, `online_activities.html`, `statements.html`)**: Auxiliary pages for user recovery, legal notices, audit logs, and document downloads.

## Design Ethos (The "Pro Max" Standard)
- **Primary Font**: Montserrat (distinct geometric sans-serif for stable corporate feel).
- **Core Accents**: Stark Brand Red (`#ed1c24`) for major triggers, deep Brand Navy (`#26247b`) for grounding structures, and Iconoir open-source icons for clean lines.
- **Surfaces**: Gentle border radiuses (`15px` cards, `8px` inputs) with floating box-shadows (`0 10px 30px rgba(0,0,0,0.2)`) and elegant glassmorphism.
